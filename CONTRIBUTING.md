# Contributing to Software Architecture Course

Thank you for your interest in contributing to the Software Architecture Course! This is an educational project designed to teach modern software architecture principles through hands-on implementation.

## How to Contribute

### For Students
1. **Fork the repository** to your GitHub account
2. **Clone your fork** locally
3. **Create a feature branch** for your work
4. **Implement your changes** following the project structure
5. **Test your changes** thoroughly
6. **Submit a pull request** to the main repository

### For Instructors
1. **Create a feature branch** from the appropriate architecture branch
2. **Implement new features or improvements**
3. **Update documentation** as needed
4. **Create pull requests** for review

## Code Guidelines

### General
- Follow the existing code style and structure
- Write clear, descriptive commit messages
- Include tests for new functionality
- Update documentation when changing behavior

### Architecture-Specific
- **SOA**: Follow centralized ESB patterns
- **MVC**: Maintain clear separation of concerns
- **Microservices**: Implement independent, loosely coupled services

### Language-Specific
- **Java**: Follow Spring Boot conventions
- **.NET**: Follow ASP.NET Core conventions
- **Node.js**: Use TypeScript and Express.js patterns
- **Python**: Use FastAPI and SQLAlchemy patterns

## Pull Request Process

1. **Ensure tests pass** - Run all relevant tests before submitting
2. **Update documentation** - Update README, guides, or ADRs as needed
3. **Describe changes** - Provide clear description of what was changed and why
4. **Link to issues** - Reference any related issues or requirements
5. **Request review** - Tag appropriate reviewers

Please be respectful and constructive in all interactions. This is an educational project intended to help people learn software architecture.
